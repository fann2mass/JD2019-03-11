тут примеры для двух вариантов DAO.

1) Простой. В пакете custom_dao лежит DAO который ориентирован на предметную область.
     при изменениях в предметной области будет меняться и сам DAO.
        Плюс: гибкость запросов SQL.
        Минус - объем кода.
     Запускается пример из V1_CustomRunner.

2) Сложный. В пакете generics_dao лежит DAO который ориентирован на предметную область.
     при изменениях в предметной области DAO не меняется, но beans работают только на основных типах данных.
        Плюс: при добавлении beans DAO не меняется.
        Минус - дополнительные требования к beans.
     Запускается пример из V2_UniversalRunner.
